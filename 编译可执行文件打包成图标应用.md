# 一、Go 编译基础

假设你的 Go 项目在 `cmd/app/main.go`，先编译 exe：

```powershell
# Windows 本机编译
go build -trimpath -ldflags "-s -w" -o build\app.exe ./cmd/app
```

* `-trimpath`：去掉绝对路径
* `-ldflags "-s -w"`：去掉符号表和调试信息，减小体积
* `-o build\app.exe`：输出 exe 文件

> 这时候 exe 已经能运行，但还没有图标和应用信息。

# 二、给 exe 添加图标和应用信息

Go 本身不支持在源码中直接添加 exe 图标，需要用 **Windows 资源文件 (.syso)**。

## 1. 使用 `rsrc` 工具（官方推荐）

`rsrc` 可以把图标和版本信息打包到 exe 中。

### 安装 rsrc

```powershell
go install github.com/akavel/rsrc@latest
```

### 创建资源文件

假设你有图标 `icon.ico`：

```powershell
# -ico 指定图标
# -manifest 可选，指定 Windows manifest 文件
# -o 指定生成 .syso
rsrc -ico icon.ico -manifest app.manifest -o rsrc.syso
```

* 生成的 `rsrc.syso` 文件会被 Go 自动链接到 exe
* 支持多分辨率 ico（16x16、32x32、256x256）

### 示例 Windows Manifest（可选）

`app.manifest`：

```xml
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<assembly xmlns="urn:schemas-microsoft-com:asm.v1" manifestVersion="1.0">
  <assemblyIdentity version="1.0.0.0" processorArchitecture="X86" name="OpenWRTCustomTools" type="win32"/>
  <description>OpenWRT Custom Tools</description>
  <trustInfo xmlns="urn:schemas-microsoft-com:asm.v3">
    <security>
      <requestedPrivileges>
        <requestedExecutionLevel level="asInvoker" uiAccess="false"/>
      </requestedPrivileges>
    </security>
  </trustInfo>
</assembly>
```

---

## 2. 编译带资源的 exe

```powershell
go build -trimpath -ldflags "-s -w" -o build\app.exe ./cmd/app
```

* Go 会自动把 `rsrc.syso` 编译进 exe
* 右键 exe → 属性 → 详细信息，可以看到应用名称、公司、版本号
* exe 也会显示你的图标

# 四、可选：单文件 exe + 配置内嵌

如果你不想生成安装程序，只希望 **单文件 exe**：

* Go 1.16+ 支持 `embed.FS` 内嵌配置文件、静态资源
* 打包成 exe 后直接运行，不依赖外部文件夹

示例：

```go
//go:embed configs/config.yaml
var defaultConfig []byte
```

然后用 `viper.ReadConfig(bytes.NewBuffer(defaultConfig))` 加载配置。

这样部署就是：

```text
app.exe
```

即可运行。

# 一、App Bundle 基本结构

macOS 的应用本质是一个 **目录结构**，通常叫 `.app`，比如：

```
MyApp.app/
├─ Contents/
│   ├─ MacOS/             <-- 可执行文件放这里
│   │   └─ myapp
│   ├─ Resources/         <-- 图标、静态资源等
│   │   └─ myapp.icns
│   └─ Info.plist         <-- 应用信息文件
```

* `Contents/MacOS/myapp` → Go 编译输出
* `Contents/Resources/myapp.icns` → 图标
* `Contents/Info.plist` → 应用名称、版本、图标等元信息

> 注意 macOS 图标必须是 `.icns` 格式。

---

# 二、编译 Go 可执行文件

假设你的 Go 项目在 `cmd/app/main.go`：

```bash
# 编译 macOS x86_64
GOOS=darwin GOARCH=amd64 go build -trimpath -ldflags "-s -w" -o build/myapp ./cmd/app
```

或者 Apple Silicon:

```bash
GOOS=darwin GOARCH=arm64 go build -trimpath -ldflags "-s -w" -o build/myapp ./cmd/app
```

> 可选：也可以做 **universal binary**（同时支持 Intel + ARM）。

---

# 三、创建 App Bundle 目录

```bash
mkdir -p MyApp.app/Contents/MacOS
mkdir -p MyApp.app/Contents/Resources
```

### 1. 拷贝可执行文件

```bash
cp build/myapp MyApp.app/Contents/MacOS/
chmod +x MyApp.app/Contents/MacOS/myapp
```

### 2. 添加图标

1. 准备一个 PNG 或 SVG 图标
2. 转换为 `.icns`（macOS 专用图标格式）

```bash
iconutil -c icns icon.iconset -o MyApp.app/Contents/Resources/myapp.icns
```

* `icon.iconset` 是包含不同尺寸 PNG 的目录
* 生成 `.icns` 文件放入 `Resources`

### 3. 创建 Info.plist

`MyApp.app/Contents/Info.plist` 示例：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple Computer//DTD PLIST 1.0//EN"
 "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>CFBundleExecutable</key>
    <string>myapp</string>
    <key>CFBundleIdentifier</key>
    <string>com.example.myapp</string>
    <key>CFBundleName</key>
    <string>MyApp</string>
    <key>CFBundleVersion</key>
    <string>1.0.0</string>
    <key>CFBundleIconFile</key>
    <string>myapp.icns</string>
    <key>LSMinimumSystemVersion</key>
    <string>10.12</string>
    <key>CFBundlePackageType</key>
    <string>APPL</string>
</dict>
</plist>
```

关键字段：

* `CFBundleExecutable` → 你的 Go 二进制文件名
* `CFBundleIconFile` → `.icns` 文件名（无需写路径）
* `CFBundleIdentifier` → 唯一标识符
* `CFBundleVersion` → 版本号

# 四、可选：自动打包脚本

```bash
#!/bin/bash
APP_NAME="MyApp"
BUILD_DIR="build"
GO_FILE="./cmd/app/main.go"

# 1. 编译 Go
GOOS=darwin GOARCH=amd64 go build -trimpath -ldflags "-s -w" -o $BUILD_DIR/$APP_NAME $GO_FILE

# 2. 创建 Bundle
mkdir -p $APP_NAME.app/Contents/MacOS
mkdir -p $APP_NAME.app/Contents/Resources

cp $BUILD_DIR/$APP_NAME $APP_NAME.app/Contents/MacOS/
chmod +x $APP_NAME.app/Contents/MacOS/$APP_NAME

# 3. 添加图标
cp icon.icns $APP_NAME.app/Contents/Resources/

# 4. 创建 Info.plist
cat > $APP_NAME.app/Contents/Info.plist <<EOL
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple Computer//DTD PLIST 1.0//EN"
 "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>CFBundleExecutable</key>
    <string>$APP_NAME</string>
    <key>CFBundleIdentifier</key>
    <string>com.example.$APP_NAME</string>
    <key>CFBundleName</key>
    <string>$APP_NAME</string>
    <key>CFBundleVersion</key>
    <string>1.0.0</string>
    <key>CFBundleIconFile</key>
    <string>icon.icns</string>
    <key>CFBundlePackageType</key>
    <string>APPL</string>
</dict>
</plist>
EOL

echo "App bundle $APP_NAME.app created!"
```

执行后会生成完整的 `MyApp.app`，可以直接双击打开，Dock 会显示图标。

# 五、注意事项

1. **Go Web 应用**

    * 如果是 CLI 或 Web 服务器，建议 `Info.plist` 里加 `LSBackgroundOnly` 或 `Terminal` 配置，保证在终端打开或后台运行。

```xml
<key>LSBackgroundOnly</key>
<string>1</string>
```

2. **图标尺寸**

    * 推荐 `.iconset` 包含 16x16, 32x32, 128x128, 256x256, 512x512 PNG，生成 `.icns`
3. **Universal Binary**（Intel + ARM）

```bash
GOARCH=amd64 go build -o build/myapp-amd64 ./cmd/app
GOARCH=arm64 go build -o build/myapp-arm64 ./cmd/app
lipo -create -output build/myapp build/myapp-amd64 build/myapp-arm64
```

4. **单文件部署**

    * 使用 Go 1.16+ `embed.FS` 打包静态文件、配置、模板，这样 `MyApp.app` 就是完整单文件包

# 三、创建 AppDir 结构

AppImage 要求目录结构类似：

```
AppDir/
├─ AppRun                <-- 启动脚本或二进制
├─ usr/
│   ├─ bin/
│   │   └─ app          <-- 你的 Go 可执行文件
│   └─ share/
│       └─ icons/hicolor/256x256/apps/app.png
│       └─ applications/app.desktop
```

### 1. 创建目录

```bash
mkdir -p AppDir/usr/bin
mkdir -p AppDir/usr/share/icons/hicolor/256x256/apps
mkdir -p AppDir/usr/share/applications
```

### 2. 复制可执行文件

```bash
cp build/app AppDir/usr/bin/
chmod +x AppDir/usr/bin/app
```

### 3. 添加图标

```bash
cp icon.png AppDir/usr/share/icons/hicolor/256x256/apps/app.png
```

### 4. 创建 `.desktop` 文件

`AppDir/usr/share/applications/app.desktop` 内容示例：

```desktop
[Desktop Entry]
Name=OpenWRT Custom Tools
Exec=app
Icon=app
Type=Application
Terminal=true
Categories=Utility;
```

* `Exec=app` 会在 AppRun 中启动你的 Go 程序
* `Terminal=true` 表示在终端打开（如果你的程序是 CLI）

# 四、添加 AppRun 文件

AppImage 的入口是 `AppRun`，可简单写：

#!/bin/bash

然后赋可执行权限：

```bash

```
