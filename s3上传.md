# 二、性能对比（真实 CI/CD 场景）

上传 **10,000 个小文件**

| 客户端     | 时间              |
| ------- | --------------- |
| AWS CLI | ~8–12 分         |
| rclone  | ~3–6 分          |
| s5cmd   | **~30 秒 – 2 分** |

原因：

```
awscli → 单线程
s5cmd → 高并发 goroutine
```

使用 **AWS CLI** 操作 **Cloudflare R2** 时，方法与 **Amazon S3** 基本一致，因为 R2 完全兼容 **S3 API**。关键是 **必须指定 R2 endpoint**。

R2 endpoint 格式：

```bash
https://<ACCOUNT_ID>.r2.cloudflarestorage.com
```

下面给你一套 **CI/CD 和脚本常用的 CRUD 命令**。

---

# 1 上传文件（Create / Upload）

上传单个文件：

```bash
aws s3 cp ./test.txt \
s3://my-bucket/test.txt \
--endpoint-url https://<ACCOUNT_ID>.r2.cloudflarestorage.com
```

上传目录：

```bash
aws s3 cp ./dist \
s3://my-bucket/dist \
--recursive \
--endpoint-url https://<ACCOUNT_ID>.r2.cloudflarestorage.com
```

---

# 2 搜索 / 判断文件是否存在

### 方法1（推荐）：head-object

```bash
aws s3api head-object \
--bucket my-bucket \
--key test.txt \
--endpoint-url https://<ACCOUNT_ID>.r2.cloudflarestorage.com
```

结果：

* **存在** → 返回 metadata
* **不存在** → `404 Not Found`

脚本判断示例：

```bash
if aws s3api head-object \
  --bucket my-bucket \
  --key test.txt \
  --endpoint-url https://<ACCOUNT_ID>.r2.cloudflarestorage.com \
  2>/dev/null
then
  echo "file exists"
else
  echo "file not found"
fi
```

---

### 方法2：list objects

```bash
aws s3 ls \
s3://my-bucket/test.txt \
--endpoint-url https://<ACCOUNT_ID>.r2.cloudflarestorage.com
```

如果存在：

```
2026-03-13 10:00:00  120 test.txt
```

# 4 更新文件（Update）

S3 / R2 **没有独立 update API**，更新就是 **重新上传覆盖**：

```bash
aws s3 cp ./test.txt \
s3://my-bucket/test.txt \
--endpoint-url https://<ACCOUNT_ID>.r2.cloudflarestorage.com
```

如果 key 相同：

```
旧对象 → 被新对象替换
```

# 8 完整 CRUD 示例脚本

```bash
#!/bin/bash

ENDPOINT=https://ACCOUNT_ID.r2.cloudflarestorage.com
BUCKET=my-bucket
FILE=test.txt

# upload
aws s3 cp $FILE s3://$BUCKET/$FILE --endpoint-url $ENDPOINT

# check exists
if aws s3api head-object \
--bucket $BUCKET \
--key $FILE \
--endpoint-url $ENDPOINT 2>/dev/null
then
  echo "file exists"
fi

# update
echo "update file"
aws s3 cp $FILE s3://$BUCKET/$FILE --endpoint-url $ENDPOINT

# delete
aws s3 rm s3://$BUCKET/$FILE --endpoint-url $ENDPOINT
```
