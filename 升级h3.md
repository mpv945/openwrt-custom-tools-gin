在 Gin 中开启 **HTTP/3 (H3)**，即基于 QUIC 的协议，目前仍然需要借助第三方库，因为即使在较新的 Go 版本中，标准库 `net/http` 主要还是 **HTTP/1.1 + HTTP/2**。

最成熟的方案是使用 **quic-go** 提供的 **HTTP/3 Server**。

Gin 本质是 `http.Handler`，因此可以直接挂到 HTTP/3 Server 上。

---

# 一、安装 HTTP/3 依赖

```bash
go get github.com/quic-go/quic-go
go get github.com/quic-go/quic-go/http3
```

---

# 二、Gin + HTTP/3 示例

```go
package main

import (
	"fmt"
	"log"

	"github.com/gin-gonic/gin"
	"github.com/quic-go/quic-go/http3"
)

func main() {

	r := gin.Default()

	r.GET("/", func(c *gin.Context) {
		c.JSON(200, gin.H{
			"message": "Hello HTTP/3",
		})
	})

	server := http3.Server{
		Addr:    ":8443",
		Handler: r, // Gin router
	}

	fmt.Println("HTTP/3 server running on https://localhost:8443")

	err := server.ListenAndServeTLS("server.crt", "server.key")
	if err != nil {
		log.Fatal(err)
	}
}
```

# 三、生成 TLS 证书（开发环境）

HTTP/3 **必须使用 TLS**。

生成测试证书：

```bash
openssl req -x509 -newkey rsa:4096 \
-keyout server.key \
-out server.crt \
-days 365 \
-nodes
```

# 五、HTTP/1 + HTTP/2 + HTTP/3 同时支持（推荐）

生产环境通常同时开启：

* HTTP/1.1
* HTTP/2
* HTTP/3

示例：

```go
package main

import (
	"log"
	"net/http"

	"github.com/gin-gonic/gin"
	"github.com/quic-go/quic-go/http3"
)

func main() {

	r := gin.Default()

	r.GET("/", func(c *gin.Context) {
		c.String(200, "Hello Gin HTTP/3")
	})

	go func() {

		// HTTP1/2
		err := http.ListenAndServeTLS(
			":8443",
			"server.crt",
			"server.key",
			r,
		)

		if err != nil {
			log.Fatal(err)
		}

	}()

	// HTTP3
	server := http3.Server{
		Addr:    ":8443",
		Handler: r,
	}

	log.Fatal(server.ListenAndServeTLS(
		"server.crt",
		"server.key",
	))
}
```

这样浏览器会：

```
HTTP/1.1 -> HTTP/2 -> HTTP/3
自动升级
```

# 七、生产环境建议

HTTP/3 通常不直接在 Gin 上启用，而是通过反向代理：

常见架构：

```
Browser
   │
HTTP/3
   │
Reverse Proxy
   │
HTTP/1.1 / HTTP2
   │
Gin
```

例如：

* NGINX
* Caddy
* Envoy

优点：

* TLS 管理
* QUIC 优化
* 更稳定

---

# 八、Gin 直接启用 HTTP/3 的优缺点

| 方案             | 优点   | 缺点   |
| -------------- | ---- | ---- |
| Gin + quic-go  | 纯 Go | 生态较新 |
| Nginx/Caddy 代理 | 稳定   | 多一层  |
| Cloudflare     | 全球优化 | 依赖云  |
